git --version : to check the version of git
git status : to check the status of the repository
git init : to initialize the repository

commits are like checkpoints in the history of the repository
git add . : to add the changes to the staging area
git commit -m "message" : to commit the changes to the repository
git log : to see the history of the repository
git log --oneline : to see the history of the repository in a single line

keep commits centric to one feature or one bug fix or one component or one file or one test case or one documentation or one refactor or one chore or one test or one build or one release or one deploy or one anything else that is related to the project.

git config --global user.name "your name" : to set the name of the user
git config --global user.email "your email" : to set the email of the user

git config --global core.editor "code --wait" : to set the editor of the git

use git ignore generator to generate the git ignore file for the project
https://www.toptal.com/developers/gitignore/api/node
